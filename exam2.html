Module 4: Network Attacks

Goal: Recognize attacks on networks that target confidentiality, integrity, and availability, and identify attacks targeting wireless networks.

1. What is a Computer Network?

A network connects computers and devices to share resources using common protocols over digital media. The Internet is essentially "Inter-Networking" (connecting LANs via WANs).

  • Devices: Laptops, desktops, smartphones, printers, external storage, cameras, and game consoles.
  • IP Addresses: Used for routing.
    • Public IP: Assigned by the ISP (e.g., 82.10.250.19).
    • Private IP: Used internally within the LAN (e.g., 192.168.0.1 for the router).
  • MAC Addresses: Media Access Control addresses reside on network adapters for local hardware communication.
  • Routers vs Switches: Routers forward data packets between different computer networks. Switches connect devices on a single network.

2. Essential Protocols & Acronyms

  • TCP/IP: Transmission Control Protocol / Internet Protocol.
  • DNS: Domain Name System protocol (resolves names to IPs).
  • SMTP: Simple Mail Transfer Protocol.
  • ARP: Address Resolution Protocol (resolves IPs to MAC addresses).
  • ICMP: Internet Control Message Protocol (used for ping/echo requests).

3. The 7 Layers of the OSI Model

  • 7. Application: Interfaces with user applications. Protocols: HTTP, SMTP, DNS.
  • 6. Presentation: Formats and encrypts data for applications.
  • 5. Session: Manages communication sessions between devices.
  • 4. Transport: Ensures reliable data transfer. Protocols: TCP, UDP.
  • 3. Network: Routes data across networks. Uses IP addressing. Device: Router.
  • 2. Data Link: Direct data transfer. Uses MAC addresses. Devices: Switches, WAPs.
  • 1. Physical: Hardware and transmission. Cables, cellular signals.

4. Primary Attack Types & The CIA Triad

  • Interception Attacks (Confidentiality/Integrity): MITM (Man-in-the-Middle), MITB (Man-in-the-Browser).
  • Poisoning Attacks (Integrity):
    • Layer 2: ARP poisoning, MAC cloning, MAC flooding (floods switch).
    • Layer 7: DNS poisoning and DNS hijacking (pharming).
  • DoS/DDoS Attacks (Availability):
    • Layer 3: Smurf Attack (ICMP echo requests).
    • Layer 4: SYN Flood Attack (TCP handshake).
    • Layer 7: HTTP flood, Slowloris, Yo-Yo attacks.

5. Wireless Attacks

  • WPAN (Bluetooth/NFC/RFID): Bluejacking (spamming), Bluesnarfing (data theft). NFC eavesdropping. RFID unauthorized tag access.
  • WLAN (Wi-Fi): Rogue Access Point (insider), Evil Twin (attacker).
  • Wireless DoS: RF jamming, dissociation attacks.

Real-World Use Scenarios

Scenario 1: The Coffee Shop Decoy (Evil Twin & MITM)

Sarah sits down at a local cafe and connects her laptop to "Cafe_Guest_Free". Unbeknownst to her, a hacker sitting in the corner has set up a malicious Wireless Access Point with that exact name. Because her Wi-Fi traffic now routes through the attacker's machine, they execute a Man-in-the-Middle (MITM) interception attack, capturing her unencrypted web traffic before forwarding it to the real internet.

Scenario 2: Black Friday Blackout (SYN Flood)

An e-commerce website is preparing for its biggest sale of the year. Suddenly, the web server stops responding to legitimate customers. An attacker is exploiting the Layer 4 TCP three-way handshake by sending thousands of SYN packets. The server responds with SYN-ACKs and waits for the final ACK that never comes, exhausting its memory and causing a devastating Availability failure.

Scenario 3: The Insider Intercept (ARP Poisoning)

An angry employee plugs a laptop directly into the corporate LAN. They run a script that broadcasts fake MAC addresses to the local switch, claiming that their laptop is the default gateway (router). The switch's MAC table is poisoned. Now, whenever a coworker tries to send a file to the internet, the Layer 2 traffic is silently redirected through the attacker's laptop first.

Module 5: Network Defenses

Goal: Learn the types of security appliances, secure access technologies, physical security measures, and wireless defense mechanisms.

1. Firewall Foundations

Firewalls utilize Access Control Lists (ACLs) to manage incoming and outgoing traffic based on the concept of Implicit Deny. They can operate at multiple OSI Layers: Layer 3 (Network/IP), Layer 4 (Transport/Ports), or Layer 7 (Applications).

Components of a rule-based firewall rule:

  • Source / Destination IP
  • Port & Protocol
  • Permission
  • Time & Context

2. Types of Firewalls

  • Stateless Firewalls: Fast and efficient, but do not track active sessions. Example: Router ACLs.
  • Stateful Firewalls: Highly effective at stopping complex threats (like SYN floods) because they track the state of active connections for legitimate sessions (e.g., outbound/inbound HTTP). Example: Windows Defender Firewall.
  • Hardware vs. Software: Hardware firewalls are dedicated physical devices (e.g., Cisco firewalls connected directly to a router). Software firewalls run as services on existing endpoints.
  • WAF (Web Application Firewall): Operates strictly at Layer 7 to protect against common web exploits. Example: Amazon WAF.

3. IDPS & Network Adjustments

  • IDPS: Intrusion Detection and Prevention Systems provide deeper packet inspection and behavioral analysis than standard firewalls. This allows them to detect and block sophisticated attacks. An IPS is essentially a Next-Generation Firewall (NGFW).
  • Network Adjustments & Defenses:
    • Load Balancing: Scaling resources to handle traffic loads safely.
    • Rate Limiting: Controlling properties like TCP/IP connections and maximum bandwidth.
    • Blackholing: Dropping malicious traffic quietly without alerting the sender.

4. Deception Technologies

Deception technologies act as "bait" for threat actors.

  • Honeypots: Decoy systems (like decoy databases, email traps, spider honeypots, malware honeypots). They are used for:
    • Neutralizing an attacker's active reconnaissance.
    • Learning the attacker's methods.
    • Distracting the attacker from main networks by providing an "illusion of hacking success."
  • Honeynet: A coordinated network of multiple honeypots.

5. Secure Access: Proxy Servers & VPNs

  • Proxy Server: Sits directly between endpoints. It hides endpoint IP addresses, providing Confidentiality (but it does NOT guarantee Integrity). Can be manual or auto-detected via DNS/DHCP. Example: EZproxy for the UT Library.
  • VPN (Virtual Private Network): Creates an encrypted tunnel over a public network. Provides both Confidentiality and Integrity, mitigating MITM attacks.
    • Remote Access VPN: Like Pulse Secure, used for individual users.
    • Site-to-Site VPN: Connects entire networks, mainly used to protect an extranet.
    • Can use full or split tunnelling.

6. NAC, DLP, and Secure Network Design

  • NAC (Network Access Control): Examines the current state of a device before it can connect. Uses permanent or dissolvable agents. Can be embedded in Windows Active Directory (AD).
  • DLP (Data Loss Prevention): Utilizes a server and rules. DLP actions include: Block, Redirect, Quarantine, and Alert.
  • Network Segmentation:
    • VLANs: Logical grouping to isolate networks (Note: VLAN segmentation failed during the Target data breach).
    • DMZ (Demilitarized Zone): A perimeter network that is external-facing to host public servers securely.
    • Air-Gapping: True, complete physical isolation of a system.

7. Securing Wi-Fi & Physical Controls

  • Wi-Fi Security:
    • WEP: Old protocol using RC4. Can cause a "detectable pattern" for attackers.
    • WPA: Uses TKIP and MICs (Message Integrity Checks).
    • WPA Personal vs Enterprise: Personal uses a pre-shared key (home routers). Enterprise uses a RADIUS server (e.g., Edu Roam).
    • WPA2 & WPA3: WPA2 uses AES. WPA3 uses AES with longer keys and features forward secrecy.
  • Physical Controls:
    • External: Barriers, security personnel, sensors.
    • Internal: Mantrap, locks, secure areas, protected cables, fire suppression.
    • Hardware: Cable locks, Faraday cages (blocks electromagnetic fields).

Real-World Use Scenarios

Scenario 1: Safe Travels (Remote Access VPN)

David is an HR executive traveling abroad for a conference. He needs to approve payroll from his hotel room, but the hotel Wi-Fi is unsecure. He opens his Pulse Secure client and connects to the company's Remote Access VPN. This creates an encrypted tunnel over the public network, ensuring that even if someone is eavesdropping on the hotel network, they only see scrambled ciphertext.

Scenario 2: The Decoy Database (Honeypot)

A threat actor breaches the outer perimeter of a corporate network and begins scanning for vulnerable databases. They find a server labeled "Customer_Financials_Backup" and successfully extract the tables. However, the security team is immediately alerted. The server was actually a Honeypot filled with fake data, designed to distract the attacker and waste their time while the security team locks down the real production database.

Scenario 3: Stopping the Web Exploit (WAF & Stateful Firewall)

A hacker attempts to breach a university's web server by injecting a malicious SQL command into the login page. The standard Layer 3 router allows the traffic because it operates on port 443 (HTTPS). However, the Web Application Firewall (WAF) operating at Layer 7 inspects the actual application payload, recognizes the SQL injection signature, and drops the traffic instantly.

Module 6: Cryptography

Goal: Understand cryptographic algorithms, digital certificates, PKI, and secure communication protocols.

1. Core Cryptographic Concepts

Cryptography transforms plaintext into ciphertext to provide confidentiality, integrity, authentication, and non-repudiation.

  • Hashing: One-way algorithm generating a fixed-length digest (a digital fingerprint). Verifies Integrity. Algorithms: MD5, SHA, RIPEMD.
  • Symmetric Encryption: Uses a single, identical shared key (Private Key). Fast, but key distribution is challenging. Algorithms: DES, 3DES, AES.
  • Asymmetric Encryption: Uses mathematically related Public and Private keys. Slower but secure. The Public key encrypts, while the Private key decrypts.

2. Asymmetric Algorithms & Key Exchange

  • RSA: Very secure (e.g., 2048-bit keys) but slow. Often used in conjunction with symmetric cryptography.
  • ECC (Elliptic Curve Cryptography): Equally secure as RSA but requires less computing power. Commonly used by university websites.
  • Diffie-Hellman (DH): Solves the "key exchange problem" by allowing two parties to securely establish a shared symmetric key over an unsecure channel (e.g., DHE, ECDH).
  • Perfect Forward Secrecy (PFS): Changes session keys frequently and automatically to protect past sessions against future compromises.

3. Digital Signatures & Certificates

  • Digital Signatures: The sender hashes the message, then encrypts the digest with their Private Key. This verifies message integrity and ensures non-repudiation.
  • Digital Certificates: Proves ownership of a public key. Issued by Certificate Authorities (CAs).
    • Domain Validation (DV): Verifies the identity of a web domain.
    • Extended Validation (EV): Shows the company's name in the browser.
    • Wildcards: Authenticates multiple subdomains.
  • PKI (Public Key Infrastructure): The framework that manages certificates using trust models (Hierarchical, Distributed, Bridged) and components like Active Directory Certificate Services (AD CS).

4. Secure Protocols & HTTPS

  • HTTPS Handshake: Uses SSL/TLS. It combines both cryptographic methods: it uses Asymmetric Encryption initially to securely exchange a session key, then switches to faster Symmetric Encryption for the rest of the web session.
  • Other Protocols:
    • SSH: Secure Shell for remote access.
    • IPsec: Network security working at Layer 3.
    • SRTP: Secure Real-Time Transport Protocol (for VoIP).
    • S/MIME: Secure email extensions.

5. Attacks, Steganography, & Hardware

  • Cryptographic Attacks:
    • Collision Attack: Attempting to find two different inputs that produce the exact same hash digest.
    • Downgrade Attack: Intentionally forcing two parties to negotiate weaker security parameters than they both support.
    • Side-Channel Attack: Analyzing physical metrics (like power consumption) to recover key bytes.
  • Steganography: Hiding the existence of data within images, audio, or video (e.g., digital watermarking).
  • Hardware Encryption: HSM (Hardware Security Module) is used mainly for networks. TPM (Trusted Platform Module) is used mainly for endpoints.

Real-World Use Scenarios

Scenario 1: The Undeniable Transfer (Digital Signatures)

The CEO of a bank sends an email authorizing a $10 million wire transfer. To prove she sent it, her email client hashes the message and encrypts the digest with her Private Key. When the CFO receives the email, their client decrypts the signature using the CEO's Public Key. Because it decrypts successfully, the CFO has cryptographic proof of Non-repudiation—the CEO cannot deny sending the email.

Scenario 2: Verifying the Update (Hashing)

A system administrator downloads a massive 5GB firmware update for the company's core routers. Before installing it, they want to ensure the file wasn't corrupted or injected with malware during the download. They run an MD5 hash command on the downloaded file and compare the resulting digest to the hash published on the vendor's website. The hashes match perfectly, verifying the file's Integrity.

Scenario 3: The Secure Checkout (HTTPS Handshake)

You type your credit card into an online store. Your browser and the web server initially use Asymmetric encryption (RSA or ECC) to securely exchange a temporary secret key over the open internet. Once both sides have the secret key, they seamlessly switch to Symmetric encryption (like AES) for the rest of the checkout process, because symmetric encryption is much faster for encrypting the actual session data.
